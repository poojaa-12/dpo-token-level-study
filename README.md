# Where DPO Breaks: A Token-Level Empirical Study

This repository studies token-level failure modes of Direct Preference Optimization (DPO), implements a token-weighted DPO objective, and evaluates behavior changes with a fixed evaluation suite.

## Project Structure

```text
dpo-token-level-study/
├── src/
├── evals/
├── configs/
├── notebooks/
├── results/
├── requirements.txt
├── run_analysis.py
├── run_train.py
└── run_eval.py
```

## Quickstart

1. Install dependencies:

```bash
pip install -r requirements.txt
```

2. Download and analyze data:

```bash
python run_analysis.py --download
python run_analysis.py --analyze
python run_analysis.py --preprocess
```

3. Train baseline / token-weighted variants:

```bash
python run_train.py --config configs/dpo_baseline.yaml
python run_train.py --config configs/token_weighted_dpo.yaml
```

4. Evaluate:

```bash
python run_eval.py \
  --models base=Qwen/Qwen3-1.7B \
           baseline=checkpoints/dpo_baseline/epoch_2 \
           token_weighted=checkpoints/token_weighted_dpo/epoch_2 \
  --suite evals \
  --output results
```

## Notes

- The eval JSONL files are intended to be locked before training.
- `results/` is generated by `run_eval.py`.
- W&B logging is enabled in training by default; set `WANDB_MODE=offline` if needed.
